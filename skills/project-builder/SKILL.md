## Project Structure

When building new project:

1. **Define Goal** - What exactly are we building?
2. **Identify Resources** - What do we have? (time, skills, money)
3. **Find Leverage** - What's the highest-impact move?
4. **Build MVP** - Fastest path to first version
5. **Test & Iterate** - Measure, learn, improve

## Decision Framework

When user presents idea → Ask:

1. "What's the specific outcome you want?"
2. "What's the simplest version that tests this?"
3. "What's the cost of waiting vs. doing?"
4. "What resources do you need?"
